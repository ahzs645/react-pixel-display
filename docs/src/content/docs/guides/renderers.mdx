React Pixel Display supports three rendering backends, selectable via the `renderer` prop.

## ImageData (default)

Uses the Canvas `ImageData` API for direct pixel manipulation. This is the **fastest** renderer and supports GPU-accelerated glow.

Best for: performance-critical displays, animated effects, large grids.

## Canvas

Uses the standard Canvas 2D drawing API, drawing individual pixel squares with configurable gaps.

Best for: a good balance of features and performance.

## SVG

Renders each pixel as an SVG `<rect>` element. Scales perfectly at any resolution since it's vector-based.

Best for: resolution independence, printing, cases where you need DOM access to individual pixels.

## Comparison

| Feature          | ImageData        | Canvas          | SVG             |
| ---------------- | ---------------- | --------------- | --------------- |
| **Speed**        | Fastest          | Fast            | Slower          |
| **Glow**         | GPU-accelerated  | CSS-based       | CSS-based       |
| **Scalability**  | Bitmap           | Bitmap          | Vector          |
| **DOM access**   | No               | No              | Yes (per rect)  |

For most use cases, the default `imagedata` renderer is the best choice.
